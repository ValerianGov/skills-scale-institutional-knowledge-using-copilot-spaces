# Handoff & Handover Checklist

Purpose: Reduce ambiguity during work handoffs between roles and teams.

Before handoff
- [ ] Context summary: objective, scope, and success criteria
- [ ] Known risks and open issues documented
- [ ] How to reproduce any outstanding bugs (steps, logs, links)
- [ ] Dependencies and integration points listed
- [ ] Suggested next steps and acceptance criteria

During handoff
- [ ] Walkthrough meeting scheduled (record notes or recording)
- [ ] Key contacts and owners introduced
- [ ] Required credentials or access noted (and secured)

After handoff
- [ ] New owner acknowledges receipt and understands next steps
- [ ] Tasks and follow-ups added to backlog with owners and due dates
- [ ] Any docs or runbooks updated for long-term ownership
